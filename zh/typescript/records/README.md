# 会话记录

本文件夹存储 CodeBTI 访谈记录。记录保留用户的项目描述、完整题目卡、固定答案、自适应答案、逐答案反馈、Agent 观察和最终风格推断。

访谈期间，在目标项目根目录维护一份实时的 `Recording.md` 和初始 `SPEC.md`。在问下一个问题前，每次回答后都更新 `Recording.md`。结束时，在生成 `CodeStyle.md`、`SKILL.md` 或最终 SPEC 更新前重新阅读两个文件。

访谈结束后，可以保留 `Recording.md`，或将其复制到此文件夹，每个访谈会话一个 markdown 文件。

文件名格式：

```text
YYYY-MM-DD-project-slug.md
```

示例：

```text
2026-04-22-typescript-api-gateway.md
```

## 记录规则

- 为每个固定和自适应问题记录完整的用户面对题目卡：标题、维度、场景、说明、代码示例（如果有）、选项和来源路径。
- 记录用户选择的答案及他们给出的任何简短解释。
- 随着访谈进行，记录简洁的按时间顺序的问答历史。
- 记录每次回答后给出的简短反馈。
- 当用户返回并重新选择时，记录答案变更。
- 将隐藏评分与用户面对的答案分开。
- 不记录秘密、凭证、私人令牌或无关的个人信息。
- 优先使用简洁笔记而非完整对话记录。
- 保留足够证据，使另一个 Agent 能够理解为什么生成了最终的 `CodeStyle.md`。
- 将最终的 `Recording.md` 和 `SPEC.md` 作为生成 markdown 输出的事实来源。

## 记录内容

每个会话应包括：

- 项目摘要和 SPEC intake，
- 访谈日期，
- 语言目标，
- 访谈进度，
- 完整题目卡快照，
- 按时间顺序的问答历史，
- 固定问题答案，
- 自适应问题答案，
- 给用户的简短反馈，
- 答案变更，
- 风格轴摘要，
- 模式信号，
- 未解决的假设，
- 生成的输出文件。

使用 [../../shared/records/session-record.template.md](../../shared/records/session-record.template.md) 作为起点。
